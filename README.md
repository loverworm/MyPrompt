# 我的提示词管理器

一款 Chrome 扩展，帮助你在 **Deepseek、豆包、Kimi** 等 AI 对话网站中高效管理和快速填入提示词。

## ✨ 主要功能

- **提示词管理**：新增、编辑、删除、本地永久存储
- **分类/标签**：自定义分类，多标签点选（编辑页直接切换选中状态）
- **三态切换**：正式列表、收藏夹、草稿箱自由切换，草稿↔正式互转
- **批量操作**：批量勾选、批量删除、批量迁移分类（复选框悬停有提示，操作自动记录日志）
- **搜索排序**：标题/内容/标签关键词搜索，支持7种排序方式（分类内/全局，按时间、使用次数、置顶、收藏、版本号）
- **置顶**：点击 📌/📍 按钮可将词条置顶。置顶词条会在其所属分类中排在该分类的最前面（不受时间排序影响），卡片带有浅黄色背景和左侧橙色边框，卡片右上角显示"已置顶"文字，同时会出现在浮动快捷面板中（带"📍 置顶"标签）。按钮图标会随状态改变：未置顶时显示 📌，已置顶时显示 📍。鼠标悬停时提示当前状态。
- **收藏视觉区分**：在主列表中，已收藏的词条（非置顶）会显示浅蓝色背景和左侧蓝色边框，卡片右上角显示"已收藏"文字，方便快速识别。
- **分类背景色**：在分类管理中可为每个分类设置背景色（15种预设颜色）。主列表中，非置顶且非收藏的词条将自动应用其分类的背景色，并带有同色系加深的左边框，便于快速识别不同分类的词条。
- **版本历史**：时间轴展示每个提示词的所有迭代版本，支持一键恢复任意旧版本，**删除生效版本时若存在其他历史版本会自动回退**。**恢复时，该标题下的所有旧版本会被自动取消收藏和置顶，恢复的版本本身也不带收藏/置顶状态。**
- **使用次数统计**：每次填入自动累计，高频词条（≥5次）自动出现在浮动面板中，并显示使用次数。
- **操作日志**：自动记录所有关键操作（新增/编辑/删除/收藏/草稿/批量/填入/复制/清空/导入导出等），支持按类型筛选、按时间段折叠（最近7天、8-30天前、30天前），自动记录操作时的目标网站域名。
- **浮动面板智能推荐（增强版）**：按 `Alt+1` 唤出快捷面板，展示 **收藏 + 置顶 + 高频词条（使用次数≥5）**。面板新增：
  - **排序切换**：可在"按使用次数排序"和"置顶优先排序"之间切换。
  - **搜索过滤**：实时根据标题或内容过滤词条。
  - **查看更多**：初始显示8条，可展开至20条。
  - **自动刷新提示**：数据变化时自动刷新并提示。
  - 卡片增加微边框和阴影，视觉更清晰。
- **一键填入**：自动识别当前页面输入框（适配 Lexical 富文本编辑器）
- **一键复制**：快速复制提示词内容到剪贴板
- **清空输入框**：浮动面板内置「清空」按钮，一键清空当前 AI 输入框
- **导入/导出备份**：导出文件名包含精确到秒的时间戳，导入支持选择本地 JSON 文件，**现在支持操作日志的追加或替换导入**
- **左侧导航分组折叠**：支持折叠/展开分组，状态自动记忆
- **分类/标签筛选**：顶部工具栏增加分类筛选和标签筛选下拉框
- **按钮悬停预览**：鼠标悬停在"功能说明"、"版本更新"按钮上可预览摘要。
- **在新标签页中打开管理器**：点击左侧导航栏顶部的按钮，可在独立标签页中打开管理界面，获得更大操作空间
- **弹窗高度动态适配**：自动检测浏览器实际视口高度并修正布局，彻底消除 Popup 模式下多余的幽灵滚动条
- **版本号动态同步**：底部信息区版本号自动读取 manifest.json，无需手动维护
- **词条卡片布局优化**：操作按钮组独立成行，标题超长自动截断且保留版本号/使用次数始终可见，分类与标签合并为同一行展示

## 🚀 快捷键

| 快捷键 | 作用 |
|--------|------|
| `Alt+1` | 唤出/关闭浮动快捷面板 |
| `Esc` | 关闭浮动面板（当面板打开时） |
| `Ctrl+Shift+L` | 打开完整管理器（弹窗） |

## 🌐 支持网站

- Deepseek（`*.deepseek.com`）
- 豆包（`*.doubao.com`）
- Kimi（`*.kimi.com`）

&gt; 其他网站不会自动注入浮动面板，但管理器仍可正常使用（填入功能需要手动点击输入框）。

## 📦 安装方法

1. 下载本扩展的文件夹（包含所有 `.html`、`.js`、`.json` 文件）。
2. 打开 Chrome 浏览器，进入 `chrome://extensions/`。
3. 开启右上角的"开发者模式"。
4. 点击"加载已解压的扩展程序"，选择扩展文件夹。
5. 安装完成后，浏览器工具栏会出现扩展图标。

## 💾 数据备份与恢复

- **导出**：点击扩展图标 → 左侧"导入/导出" → "导出全部数据"，文件将自动保存到下载文件夹，文件名格式为 `提示词备份_YYYY-MM-DD_HH-MM-SS.json`。** 现在导出会自动包含操作日志**。
- **导入**：点击"导入/导出" → 选择备份 JSON 文件 → 选择"追加日志"或"替换日志"模式 → 点击"确认导入"。** 支持灵活合并日志记录**。导入时自动兼容旧版分类数据格式（字符串数组 → 对象数组），缺失颜色字段会自动补充默认值，不会导致错误。
- 数据存储在 `localStorage` 和 `chrome.storage.local` 中，两者实时双向同步，确保浮动面板与管理器数据一致。

## 📝 使用提示

1. **首次使用**：在管理器中创建分类、标签，然后新增提示词。
2. **快速填入**：在支持的 AI 网页中按 `Alt+1` 打开浮动面板，点击任意词条即可自动填入输入框，使用次数自动增加，**操作自动记入日志**。
3. **智能推荐**：任何正式词条使用次数达到5次后，会自动出现在浮动面板中（即使未收藏或置顶），按使用次数降序排列。
4. **版本管理**：
   - 编辑正式词条时，版本号会自动递增（0.1→0.2），旧版本会保留在版本历史中并自动禁用。
   - 如需恢复旧版本，打开"版本历史"页面，点击对应版本的"恢复为此版本"即可。恢复时会清除同标题所有旧版本的收藏和置顶状态。
   - 在版本历史页面删除当前生效版本时，若该词条下还存在其他历史版本，系统会自动将最新历史版本恢复为生效状态，防止误删导致数据丢失。
5. **数据同步**：从浮动面板填入后，管理器界面会自动刷新次数，无需手动操作。
6. ** 查看操作日志**：点击左侧导航栏的"📋 操作日志"，可按类型筛选、展开/折叠历史记录，随时回溯你的所有操作。
7. **删除确认**：所有删除操作（包括主页面、收藏页面、草稿箱）都会弹出确认对话框，防止误删。
8. **浮动面板排序**：面板默认按使用次数从高到低排列，次数相同则最近使用过的排在前面。点击面板顶部的"次数优先"按钮可切换为"置顶优先"排序，置顶词条将排在前面（内部仍按使用次数降序）。置顶词条一定会出现在面板中（只要未被禁用）。
9. **草稿转正式**：草稿转为正式词条时，系统会自动禁用所有同名的旧正式词条（保留在版本历史中），草稿成为新的正式版本，**版本号在草稿原版本号基础上自动递增（如 0.3 → 0.4）**。
10. **视觉区分**：
    - 置顶词条：浅黄色背景+橙色左边框，卡片右上角"已置顶"文字标签。
    - 收藏词条（非置顶）：浅蓝色背景+蓝色左边框，卡片右上角"已收藏"文字标签。
    - 普通词条若所属分类有设置背景色，则显示该分类背景色+同色系加深左边框。
    - 如果一个词条同时置顶和收藏，优先显示置顶样式和"已置顶"标签。
11. **分类背景色**：进入"分类管理"页面，点击分类旁边的颜色下拉框，可为该分类选择背景色（15种预设颜色）。保存后，所有属于该分类的普通词条（非置顶且非收藏）都会显示该背景色和同色系左边框。
12. **排序功能详解**：点击顶部工具栏的排序下拉框，可选7种排序方式：
    - **默认排序**：分类内最新修改在前（置顶优先）
    - **分类内：最早修改在前**：分类内按修改时间升序（置顶优先）
    - **分类内：使用次数最多**：分类内按使用次数降序（置顶优先）
    - **全局：使用次数最多**：所有词条按使用次数降序（跨分类，不分组，忽略置顶优先）
    - **全局：置顶优先**：所有置顶词条在前，内部按使用次数降序
    - **全局：收藏优先**：所有收藏词条在前，内部按使用次数降序
    - **全局：版本号降序**：所有词条按语义版本号降序（如 2.0 → 1.5）
13. **浮动面板新功能**：打开浮动面板后，可使用顶部搜索框快速查找词条；点击"次数优先"按钮可切换为"置顶优先"排序；若显示词条较多，点击"查看更多"可展开至20条。
14. **弹窗自适应高度**：🆕 V1.7.1 起，Popup 弹窗会自动适配浏览器实际视口高度，消除多余的无法滚动的滚动条，提升界面整洁度。
15. **词条卡片布局**：🆕 第一行显示复选框、置顶按钮、标题及版本号（标题超长自动截断，版本号和使用次数始终完整可见）、状态标签；第二行显示操作按钮组；分类和标签合并为同一行展示。

## 🛠 权限说明

- `activeTab`：临时访问当前标签页，用于填入和清空输入框。
- `scripting`：向页面注入脚本以操作输入框。
- `storage`：存储提示词、分类、标签、使用次数统计以及**操作日志**等数据。

## 🙋 常见问题

**Q：为什么在百度等网站按 Alt+1 没有反应？**  
A：当前仅支持 Deepseek、豆包、Kimi 三个网站。如需添加其他网站，可修改 `manifest.json` 中的 `content_scripts.matches`。

**Q：浮动面板不显示任何词条？**  
A：请先在管理器中添加提示词，并将其"收藏"或"置顶"，或者多次使用（达到5次以上）。面板会展示收藏、置顶和高频词条。

**Q：编辑词条后版本号不变？**  
A：只有编辑**正式词条**（非草稿）并点击"保存正式"时，版本号才会自动递增并保留旧版本。草稿编辑不会改变版本号。

**Q：恢复旧版本后，收藏和置顶状态如何？**  
A：恢复旧版本时，系统会自动清除该标题下所有旧版本的收藏和置顶状态，新恢复的版本也不会自动收藏或置顶，需要您手动设置。

**Q：导入导出支持哪些数据？**  
A：导出包含全部正式词条、草稿、收藏、分类、标签、使用次数统计以及**操作日志**。导入时可选择追加或替换日志。导入时自动兼容旧版分类数据格式，不会出错。

**Q：操作日志会占用很多存储空间吗？**  
A：日志最多保存 500 条，超出后自动删除最早的记录。你也可以手动清空所有日志。

**Q：我的收藏中的"删除"按钮是彻底删除吗？**  
A：是的。在"我的收藏"页面点击"删除"按钮，会彻底删除该词条（从所有列表中移除），并弹出确认框。如果只想取消收藏，请点击"取消收藏"按钮。

**Q：草稿转为正式后，之前的正式版本会怎样？**  
A：草稿转正式时，系统会自动禁用所有同名的旧正式词条（保留在版本历史中），草稿成为新的正式版本，**版本号在草稿原版本号基础上自动递增（如 0.3 → 0.4）**。

**Q：分类背景色会与置顶/收藏冲突吗？**  
A：不会。优先级为：置顶样式 &gt; 收藏样式 &gt; 分类背景色。置顶和收藏的词条会强制使用其专用样式和右上角文字标签，分类背景色仅对普通词条生效。

**Q：全局排序时为什么没有分类标题？**  
A：全局排序旨在跨分类统一排序，因此不显示分类标题，所有词条混排成一整个列表。

**Q：标题超长时版本号会被截断吗？**  
A：不会。词条卡片采用标题与版本号分离显示，标题超长时仅截断标题文字，版本号和使用次数始终完整显示。
